Cómo funciona

El contexto se mantiene solo. Nadie tiene que acordarse de actualizar nada.

  1. El admin registra el repositorio. El hub lo clona, construye el grafo de su código y deja el webhook activo en GitHub.
  2. Cada push a la rama principal reindexa el grafo. Sin intervención: el contexto que consultan los agentes nunca queda viejo.
  3. Cada pull request recibe una verificación arquitectónica. Se cruza lo que cambió contra los lineamientos oficiales y el veredicto queda comentado en el PR.
  4. Los desarrolladores consultan por MCP. Su agente pregunta al hub — el grafo real y las reglas oficiales, no lo que el modelo supone.

Credenciales de acceso

Tokens con los que el hub clona los repositorios y registra los webhooks en GitHub.

¿Qué permisos necesita el token?

En GitHub: Settings → Developer settings → Personal access tokens → Fine-grained, con acceso a los repositorios que quieras vigilar.
